Objetivo

Introducir a los estudiantes en un esquema de desarrollo de software similar a los utilizados en la industria, utilizando buenas prácticas y tecnologías acordes a las prácticas actuales.

Buenas prácticas relacionadas a los métodos y herramientas utilizadas, el diseño, la arquitectura y el código que escribimos.

¿Qué tecnologías vemos?

Persistencia OO con Hibernate y servicios Web con SpringBoot.

Front-ends Web con React.

Si alguno prefiere otra tecnología, puede proponerla a la cátedra.

¿Qué contenidos teóricos vemos?

Persistencia OO con JPA/Hibernate. Testing de Integración.

Conceptos de Arquitectura de Software: arquitecturas monolíticas vs distribuídas, partición lógica vs física. El modelo C4.

El protocolo HTTP, Servicios Web con SpringBoot.

Front-ends Web: Server Side Rendering (SSR) vs Client-side Rendering. Single Page Applications (SPA) con Reactjs.

Autenticación y Autorización. Seguridad en aplicaciones Web (Keycloak).

¿Qué contenidos teóricos vemos?

Conceptos de Colas de Mensajes e implementación con RabbitMQ.

Conceptos de Bases de Datos NoSQL. Base datos de Clave-Valor y Documentos.

Conceptos de Integración Continua.

Y más sobre tendencias...

Trabajos Prácticos

Dos Trabajos Prácticos: Uno Individual y otro grupal.

TP Individual: Mini-Twitter. Implementación full stack con las tecnologías Web vistas en clase. Con una arquitectura simple monolítica.

TP Grupal: Ventas Online de Películas. Implementación full stack de una aplicación Web con arquitectura distribuída, utilizando las tecnologías y prácticas vistas en clase. Se trabajará en grupos de 2 o 3 alumnos.

Metodología

Los miércoles habrá clases teóricas (presencial). Los viernes consultas de TPs (virtual).

El TP Grupal se irá construyendo en iteraciones de 2/3 semanas. Sincronizado con las clases teóricas.

Luego de cada iteración se realizará una demo de lo construído.

Demo: Qué habíamos planeado vs qué construímos. Se muestra el plan (casos de uso, historias de usuario, etc) y se procede a mostrar la funcionalidad construída a partir del plan como si los docentes fueran el cliente.

Demo: Luego, también se discutirán cuestiones técnicas del desarrollo.

Metodología

Cada vez va a ser más importante escribir buenas historias de usuario. ¿Tienen idea porque?

Tip: La IA es buena escribiendo historias de usuario. Iteren con ella para armar cada sprint.

Cursada y Final

Cursada: Se aprueba con la entrega/finalización del TP Individual. Y las demos de cada iteración (mínimo 3).

Final: Finalizar el TP Grupal, más una tarea adicional (consensuada con la cátedra) individual sobre el TP Grupal.
