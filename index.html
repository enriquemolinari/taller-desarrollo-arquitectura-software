La arquitectura de software establece de qué manera un producto de software o sistema está estructurado. La estructura comprende elementos de software, sus propiedades y las relaciones entre ellos.

Llamamos elemento de software a cualquier pieza de software que podría ir desde una función, procedimiento, clase, módulo, capa y paquete hasta aplicaciones, repositorios de datos, ambientes de ejecución (web servers, contenedores, etc), API gateways, colas de mensajes, etc.
